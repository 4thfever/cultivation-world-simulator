Yêu cầu:
1. Dựa trên hồ sơ tương tác, phân tích tương tác giữa hai người như thế nào.
2. Có đáp ứng điều kiện để thiết lập quan hệ mới hoặc hủy bỏ quan hệ cũ theo quy tắc không?
3. Phân tích liệu quan hệ có nên thay đổi; việc thêm hoặc hủy quan hệ phải đáp ứng các điều kiện liên quan.

Trả về định dạng JSON:
{{
    "analysis": "...", // Phân tích ngắn về lý luận, chỉ rõ tại sao thay đổi hoặc tại sao không.
    "changed": true | false, // Có xảy ra thay đổi quan hệ hay không.
    "change_type": "ADD" | "REMOVE", // Loại thay đổi. Có thể bỏ qua khi changed là false.
    "relation": "LOVERS" | "FRIEND" | "ENEMY" | "MASTER" ... (Phải là tên enum viết hoa), // Quan hệ liên quan. Có thể bỏ qua khi changed là false. Lưu ý đây là trạng thái của {avatar_a_name} so với {avatar_b_name}. Ví dụ, nếu output là MASTER, nghĩa là A trở thành sư phụ của B.
    "reason": "..." // Mô tả ngắn lý do, một danh từ trong mười từ, ví dụ "Tri kỷ tương phùng". Có thể bỏ qua khi changed là false.
}}
