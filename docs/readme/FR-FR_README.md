## 📖 Présentation

**Cultivation World Simulator** est un simulateur de monde xianxia piloté par l'IA.
Chaque cultivateur y est un Agent indépendant, capable d'observer son environnement et de prendre ses propres décisions. Pour éviter les hallucinations et les dérives excessives, le monde est encadré par des règles de cultivation riches et cohérentes. Dans cet univers tissé par les règles et l'IA, les cultivateurs, les sectes et les volontés d'organisation rivalisent, coopèrent et produisent sans cesse de nouvelles histoires.

Vous pouvez rester un observateur lointain, voir les sectes s'élever ou s'effondrer, assister à l'ascension de génies ou à leur chute, ou encore intervenir subtilement en déclenchant une tribulation céleste, en modifiant un esprit ou en influençant le cours du monde.

### ✨ Points forts

- 👁️ **Jouer le "Dao Céleste"** : vous n'êtes pas un cultivateur, mais la volonté qui observe et infléchit les règles du monde.
- 🤖 **Tous les PNJ sont pilotés par IA** : chaque personnage possède sa personnalité, sa mémoire, ses relations et sa logique d'action. Ils aiment, haïssent, complotent, s'allient et peuvent même défier leur destin.
- 🌏 **Des règles comme fondation** : racines spirituelles, royaumes, techniques, personnalités, sectes, élixirs, armes, tournois, enchères et espérance de vie structurent un monde crédible.
- 🦋 **Des récits émergents** : il n'y a pas de scénario figé. Les guerres de sectes, les conflits entre voie droite et voie démoniaque, les révélations de destin et les accidents du monde naissent des causalités internes.

## 🚀 Démarrage rapide

### Méthode recommandée

- **Pour modifier le code ou déboguer** : utilisez le déploiement depuis les sources avec Python `3.10+`, Node.js `18+` et un service de modèle disponible.
- **Pour jouer directement** : récupérez gratuitement la version desktop sur [Epic Games Store](https://store.epicgames.com/p/ai-cultivation-world-simulator-adebb8), ou lancez le projet avec Docker.
- Pour consulter rapidement les informations du monde, les actions, les sectes et d'autres données, vous pouvez générer un wiki local dans `tools/wiki`.

### Première utilisation

- Avec la version desktop Epic, lancez le jeu puis confirmez la configuration du modèle depuis la page des paramètres avant de commencer une nouvelle partie.
- Avec les sources ou Docker, configurez d'abord un préréglage de modèle disponible, par exemple DeepSeek, MiniMax ou Ollama.
- En mode développement, la page frontend s'ouvre normalement automatiquement. Sinon, utilisez l'adresse affichée dans les logs de démarrage.

### Méthode 0 : version desktop Epic Games Store

Adaptée aux joueurs qui veulent simplement essayer le jeu sans environnement de développement.

1. Installez gratuitement le jeu depuis [Epic Games Store](https://store.epicgames.com/p/ai-cultivation-world-simulator-adebb8).
2. Au premier lancement, vérifiez la configuration du modèle dans les paramètres.
3. Si vous souhaitez utiliser votre propre service de modèle, sélectionnez un préréglage comme DeepSeek, MiniMax ou Ollama.

### Méthode 1 : lancement depuis les sources (mode développement, recommandé)

Adaptée aux développeurs et aux personnes qui veulent modifier ou inspecter le projet.

1. **Installer les dépendances et démarrer**

   ```bash
   # 1. Installer les dépendances backend
   pip install -r requirements.txt

   # 2. Installer les dépendances frontend (Node.js requis)
   cd web && npm install && cd ..

   # 3. Démarrer le service (backend + frontend)
   python src/server/main.py --dev
   ```

2. **Configurer le modèle**

   Dans la page des paramètres du frontend, choisissez un préréglage de modèle. La configuration est enregistrée dans le répertoire de données utilisateur.

3. **Accéder au frontend**

   En mode développement, le serveur frontend est lancé automatiquement. L'adresse habituelle est `http://localhost:5173`.

### Méthode 2 : déploiement Docker en une commande (non testé)

```bash
git clone https://github.com/4thfever/cultivation-world-simulator.git
cd cultivation-world-simulator
docker-compose up -d --build
```

Frontend : `http://localhost:8123`

Le conteneur backend persiste les données utilisateur avec `CWS_DATA_DIR=/data` : paramètres, secrets, sauvegardes et logs. Par défaut, ces données sont montées dans `./docker-data` sur l'hôte et survivent à un `docker compose down` suivi d'un nouveau `up`.

> ⚠️ L'interface mobile n'est pas encore entièrement adaptée. À utiliser comme aperçu.

2. **Frontend** : dans `web/vite.config.ts`, ajoutez `host: '0.0.0.0'` dans le bloc `server`.
3. **Accès** : assurez-vous que le téléphone et l'ordinateur sont sur le même Wi-Fi, puis ouvrez `http://<IP locale du PC>:5173`.

<details>
<summary><b>API externe / intégration Agent ou Claw (cliquer pour ouvrir)</b></summary>

Cette partie sert aux agents externes, aux scripts d'automatisation ou à une boucle de jeu du type "observer -> décider -> intervenir -> observer à nouveau".

Points d'entrée recommandés :

- Requêtes en lecture seule : `/api/v1/query/*`
- Écritures contrôlées : `/api/v1/command/*`

Endpoints utiles pour démarrer :

- `GET /api/v1/query/runtime/status`
- `GET /api/v1/query/world/state`
- `GET /api/v1/query/events`
- `GET /api/v1/query/detail?type=avatar|region|sect&id=<target_id>`
- `POST /api/v1/command/game/start`
- `POST /api/v1/command/avatar/*`
- `POST /api/v1/command/world/*`

Flux minimal :

1. Appeler `GET /api/v1/query/runtime/status` pour vérifier l'état courant.
2. Si aucune partie n'est lancée, appeler `POST /api/v1/command/game/start`.
3. Utiliser `world/state`, `events` et `detail` pour lire l'état du monde et la cible.
4. Exécuter une intervention via un endpoint `command`.
5. Relire l'état via `query`, sans supposer que le cache local reflète le résultat.

Réponse réussie typique :

```json
{
  "ok": true,
  "data": {}
}
```

En cas d'échec, l'API renvoie une erreur structurée. Lisez `detail.code` et `detail.message` pour gérer les cas côté programme.

Notes :

- Les paramètres de l'application restent gérés par `/api/settings*` et `/api/settings/llm*`.
- Pour la liste complète des interfaces et les conventions d'extension, consultez `docs/specs/external-control-api.md`.

</details>

### 💭 Pourquoi créer ce projet ?

Les romans de cultivation décrivent des mondes immenses, mais le lecteur n'en voit qu'un fragment.

Les jeux de cultivation existants reposent souvent soit sur un scénario prédéfini, soit sur des machines à états très simples. Cela crée facilement des comportements artificiels ou incohérents.

Avec les grands modèles de langage, l'idée que "chaque personnage puisse sembler vivant" devient beaucoup plus accessible.

Le but est de créer un monde de cultivation vivant, direct et immersif : ni simple vitrine promotionnelle, ni pure démonstration de recherche, mais un monde jouable qui donne une vraie sensation de présence.
